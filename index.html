khata-standard

The open file format and reference data source for browser-native, local-first Indian accounting.

Status: early bootstrap

This standard is under active development alongside its first reference implementation, Bahi. The format version 1.0 will be frozen once Bahi reaches beta. Some reference datasets are still placeholders — see SEED-NOTES.md before using any data for real tax calculations.

What is this?

.khata is a file format for Indian accounting records — invoices, ledgers, GST data, customers, vendors, attachments — as a single portable file on your own computer. No servers, no accounts, no sync. This repository is the authoritative source for the format specification and the reference data that any .khata implementation needs (GST rates, HSN codes, state lists, TDS/TCS sections, and more).

Reference data CDN

Applications fetch reference data from these URLs:

https://naklitechie.github.io/khata-standard/data/index.json
https://naklitechie.github.io/khata-standard/data/states/states-20250101.json
https://naklitechie.github.io/khata-standard/data/gst-rates/gst-rates-20170701.json
... and so on

The index.json file is the manifest — start there to see what's available and current versions. Every dataset file has an sha256: hash in the index; applications should verify downloaded files against these hashes before caching.

Use in Bahi

Bahi is the first reference implementation of the .khata format. Bahi fetches from the CDN URLs above and verifies data integrity via the manifest hashes. Bahi is also the driving force behind the format's evolution, but the format itself is deliberately not owned by Bahi — anyone can build a reader or writer.
